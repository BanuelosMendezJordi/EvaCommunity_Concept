Get Started

For beginners, we developed a visual guide to help you understand and get access to the physical and logical components needed.

For advanced users and developers, you can directly access to the Github repository containing the documentation and instructions.

Choosing between versions

Before starting to build your project, we should keep in mind that there is currently more than one version, so it's important to properly identify which version is the most optimal for your needs and purposes.

Eva Basic

The most basic version of the Eva social robot has three key capabilities:

  • Verbal interaction using a speaker, and a Matrix Voice with a built-in microphone array and LEDs.
  • Animated eyes, which are displayed in a 5,5" touchscreen.
  • Head movement using two Dynamixel AX-12A servomotors.
3D Printable models
  • Common Parts
  • Common Parts (Basic and Camera versions)
  • Basic Version
Eva with Camera

This version inherits the capabilities of the basic version, along with a new key capability:

  • Vision or sight, thanks to an Intel RealSense Depth Camera D435i, located at the top of the screen.
  • This version modifies certain elements to properly adapt the camera device.
3D Printable models:
  • Common Parts
  • Common Parts (Basic and Camera versions)
  • Camera Version
Eva with Camera and Movement

This version inherits the capabilities of the previous two versions (basic and with camera); however, it contains a new key capability, which is the following:

  • Movement: using two new motors/engines and wheels.
  • This version discards the small legs to provide a fitting design for the movement functions.

  • Common Parts
  • Version with Mobility
  • Camera Version

Requirements

Below you will find a list of the needed prerequisites and components to setup your initial Eva build.

Relevant components
Raspberry Pi 4B

A low cost, credit-card sized computer that has been used in a wide array of digital maker projects, from music machines and parent detectors to weather stations and tweeting birdhouses.
Matrix Voice

MATRIX Voice is a development board for building sound driven behaviors and interfaces. It has a built-in microphone array and LEDs, and has been built to be a complete, affordable, and user-friendly tool for IoT voice app creation.
Raspberry Pi OS

The official operating system for the Raspberry Pi. Based on Debian and previously called Raspbian, is required for the Raspberry Pi card to work. For Eva, it's necessary to download this Buster version.
Waveshare 5.5" Touch AMOLED Display

A 5.5inch Capacitive Touch AMOLED Display, with a 1080 x 1920 pixel resolution, HDMI and Toughened Glass Cover. It will allow to display Eva's eyes and select between a set of options that the applications offers.
Raspberry Pi Imager

Raspberry Pi Imager is the quick and easy way to install Raspberry Pi OS and other operating systems to a microSD card, ready to use with your Raspberry Pi.
Philips BT50-37 Speaker

A compact-designed speaker with advanced audio performance, a built-in rechargable battery, anti-clipping function and easy connection to almost any electronic device.

Open CR 1.0 Movement

Robotis OpenCR1.0 is an integrated open source robot controller with a powerful MCU from the ARM Cortex-M7 range, accessible and open to the public.
ArbotiX-M Basic Camera

This versatile robot controller is an advanced control solution for BIOLOID and DYNAMIXEL based actuators. As an Arduino compatible microcontroller the Arbotix Robocontroller also benefits from a huge open source community of libraries and examples.
Other components
  • Ball Caster
  • Bioloid Frame F2
  • Bioloid Frame F3
  • Dynamixel XL430-W250-T
  • Two Dynamixel AX-12A servomotors
  • GPIO Stacking header
  • Intel® RealSense™ Depth Camera D435i Camera
  • LI-PO 11.1V 1800mAh Lb-12 19.98Wh 5C
  • Rotating HDMI Male-Female Adapter
  • Two Robot wheels Movement
  • Two Robot tires Movement
  • SanDisk Extreme 64GB Micro SD Card (recommended for performance)

Assembling

  • Insert your SD card into your computer.
  • Run the Raspberry Pi Imager installer and follow the installation instructions.
  • Once finished, run the installed application.
  • In the operating system section, select Choose OS > Use Custom > Select the Raspberry Pi OS version provided in the Requirements section (should be named 2021-05-07-raspios-buster-armhf).
  • In the Storage section, click on Choose Storage > Select the corresponding SD card.
  • Once selected, click on the Write button and wait until the operating system files get written inside the SD card.
  • Remove the SD card when a pop-up window displays the indication.
  • Insert the SD card into the Raspberry Pi 4B.
  • Turn on your Eva and wait a few moments until it boots up.
  • Follow the instructions in the initial setup; it's recommended to set the following as your username and password:

    • Username: pi

    Password: raspberry

  • Make sure to connect to a wireless network during this setup in order to obtain the initial updates.
  • Once finished, you can restart your operating system.
  • Click on the start menu button
  • Go to Preferences > Raspberry Pi Configuration
  • Enable the VNC Server service.
  • Save changes and close the window.
  • Open the terminal and type the following command: ipadress
  • On your main computer, download and run VNC Viewer.
  • In the input box, type the provided IP address and press Enter.
  • If the same IP as the Raspberry was inserted, a small prompt should popup requesting your credentials. Type the credentials used during the setup configuration:

    • Username: pi

    Password: raspberry

  • You should be able to operate the Raspberry Pi OS remotely from your own machine.
  • Before installing other dependencies, type this command in your console:

    • sudo apt update

  • NodeJS 14.16.0 is needed to run Javascript applications and scripts. To install it, run the following commands in your console (one by one):

    • curl https://raw.githubusercontent.com/creationix/nvm/master/install.sh | bash

    source ~/.profile

    nvm install v14.16.0

    nvm use v14.16.0

  • Node Package Manager (NPM) is another dependency needed to run certain commands. To install it run the following command:

    • sudo apt install npm

  • Another element needed is Electron. Copy the following command to install it:

    • sudo npm i electron -g --verbose --unsafe-perm=true --allow-root

  • Once done, install the Matrix Voice drivers using the following commands:

    • curl https://s3.amazonaws.com:443/apt.matrix.one/doc/apt-key.gpg | sudo apt-key add -

    echo "deb https://apt.matrix.one/raspbian $(lsb_release -sc) main"

    sudo tee /etc/apt/sources.list.d/matrixlabs.list

  • Update and reboot with the following commands:

    • sudo apt-get update

    sudo apt-get upgrade

    sudo reboot

  • Once rebooted, run the following commands and reboot again when done:

    • sudo apt install matrixio-kernel-modules

    sudo apt-get install matrixio-creator-init libmatrixio-creator-hal libmatrixio-creator-hal-dev

    sudo reboot

  • Using the Raspberry Pi, download the AppImage and source code from here.
  • Place the downloaded files in your desired path (it's recommended to place the files inside a new folder).
  • Open your terminal to enter the following command and create a new service:

    • nano /etc/systemd/system/eva.service

  • Once ready, add the following code inside the file you just created (note: change User and ExecStart lines according to your user and directory where you put the AppImage file):

    • [Unit]

    Description=Eva

    After=network.target


    [Service]

    Type=simple

    User=USER

    LimitNOFILE=1024


    Restart=on-failure

    RestartSec=10

    startLimitIntervalSec=60

    Environment=DISPLAY=:0

    ExecStart=/PATH/TO/THE/APP.AppImage


    PermissionsStartOnly=true

    StandardOutput=syslog

    StandardError=syslog


    [Install]

    WantedBy=multi-user.target

  • Save the files and exit the editor.
  • Enable the service to be started at boot with these console commands:

    • systemctl enable eva.service

    systemctl --system daemon-reload

  • The following step is to install pm2, a process manager that will help in keeping the Eva web application online to access and control Eva's main functions from your browser:

    • npm install pm2 -g

  • The following commands will enable the Eva service for online access from any device connecter to the same network:

    • pm2 start ./app.js --name eva

    pm2 startup

    sudo env PATH=$PATH:/home/pi/.config/nvm/versions/node/v14.16.0/bin /home/pi/.config/nvm/versions/node/v14.16.0/lib/node_modules/pm2/bin/pm2 startup systemd -u pi --hp /home/pi

    pm2 save

  • Now that everything is done, you should be able to access the web application by entering the robot's IP in your browser.
  • There are certain services that need to be enabled for Eva to work with all the available functions. To enable them, open your browser and access Eva using the provided IP, then go to the Services tab.
  • An empty list should be displaying all the available services but with empty values. To proceed, input the following values inside the corresponding variables:
    • Variable Value
    TEXT_TO_SPEECH_APIKEY tts-api-key
    TEXT_TO_SPEECH_URL https://stream.watsonplatform.net/text-to-speech/api
    SPEECH_TO_TEXT_APIKEY stt-api-key
    SPEECH_TO_TEXT_URL https://stream.watsonplatform.net/speech-to-text/api
    TRANSLATOR_APIKEY translator-api-key
    TRANSLATOR_URL https://api.us-south.language-translator.watson.cloud.ibm.com/
    GOOGLE_APPLICATION_CREDENTIALS credencial.json
    DIALOGFLOW_PROJECT_ID google-dialogflow-proyect-name
  • If the values were entered correctly, a green check icon should display.
  • If your screen is displaying vertically, use VNC Viewer in your machine and access Eva remotely
  • Click on the Raspberry Pi OS task bar button > Preferences > Screen Configuration
  • On the Screen Configuration window, click on the Configure > Screens > HDMI 1 > Orientation > Left
  • After a reboot, the screen should be displaying horizontally, but the touch function might not be synchronized with the displayed orientation.
  • To fix this, open your terminal and enter this command:

    • cd /usr/share/X11/xorg.conf.d/

    ls

  • Locate the file named d/40-libinput.conf and open it using this command

    • sudo nano 40-libinput.conf

  • Once opened, search for the section that says "Identifier... Touchscreen catchall"
  • Add the following line at the end of the section:

    • Option "TransformationMatrix" "0 -1 1 1 0 0 0 0 1"

  • Save your changes and reboot your system. After reeboting, the screen and touchscreen should have the same orientation.
  • It's possible that when opening your Eva web application, the "Interaction" section may not be displaying correctly. This is due to the Visual Programming Language needing certain files from the GoJs Framework. Right click on the "Interaction" page and select the "Inspect" option to display the errors related with this issue.
  • Access the GoJs website to explore the available documentation and scripts.
